# DataMind V1.0 交付前全面 Bug 检查计划

## 目标
在将 DataMind V1.0 交付给用户真实使用前，进行全面的 bug 检查，确保软件稳定可靠。

## 检查维度

### 1. 打包与安装验证
- [ ] NSIS 安装包能正常安装到默认路径
- [ ] 便携版能正常解压运行
- [ ] 安装后桌面快捷方式正确创建
- [ ] 卸载程序能正常卸载并清理残留
- [ ] 安装后首次启动无白屏

### 2. 核心功能验证
- [ ] Excel 文件拖拽上传正常
- [ ] 数据预览显示正确（行列数、日期检测）
- [ ] 字段映射功能正常（自动匹配、手动选择）
- [ ] 时间筛选功能正常（基于数据实际时间）
- [ ] 四个模板（Top10/产品/对比/地区）都能正常生成报告
- [ ] PDF 导出功能正常
- [ ] Excel 导出功能正常

### 3. 数据准确性验证
- [ ] 销售额计算准确（与原始 Excel 对比）
- [ ] 排名计算正确
- [ ] 占比计算正确
- [ ] 日期筛选结果正确
- [ ] 智能单位显示正确（小数值不丢失精度）

### 4. 边界情况验证
- [ ] 空 Excel 文件处理（友好提示）
- [ ] 大数据量（5万行）处理正常
- [ ] 超大文件（>15万行）正确拒绝
- [ ] 特殊字符文件名处理
- [ ] 日期格式混乱数据处理

### 5. UI/UX 验证
- [ ] 步骤导航器正常显示和跳转
- [ ] 各页面布局无错位
- [ ] 图表正常渲染
- [ ] 中文显示正常无乱码
- [ ] 窗口大小调整时布局自适应

### 6. 错误处理验证
- [ ] 无效 Excel 文件有错误提示
- [ ] 字段映射不完整有验证提示
- [ ] 保存文件时权限不足有提示
- [ ] 网络无关功能（完全离线可用）

### 7. 性能验证
- [ ] 1万行数据加载 < 3秒
- [ ] 5万行数据加载 < 10秒
- [ ] 报告生成 < 5秒
- [ ] 界面操作无卡顿

### 8. 配置与数据持久化
- [ ] 用户设置能正确保存
- [ ] 映射历史能正确保存
- [ ] 重启后设置保留

## 检查方法
1. 代码审查：检查关键文件是否有明显问题
2. 功能测试：手动执行各功能点
3. 边界测试：使用极端数据测试
4. 日志检查：查看控制台是否有错误

## 关键检查文件清单
- electron/preload.js - ESM 语法正确性
- electron/main.js - 路径逻辑正确性
- electron-builder.yml - 打包配置正确性
- dist/index.html - 资源路径为相对路径
- src/stores/mapping.js - 状态管理逻辑
- src/utils/fileHandler.js - 文件处理逻辑
- src/templates/*/calculator.js - 计算逻辑

## 交付标准
- 所有检查项通过
- 无控制台错误
- 用户操作流程顺畅
- 数据计算准确
