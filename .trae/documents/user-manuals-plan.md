# DataMind V1.0 用户说明书编写计划

## 目标

编写两份用户说明书：

1. **安装说明书**：手把手教用户安装软件，重点解决"管理员权限提示"问题
2. **使用说明书**：手把手教用户使用，重点解决"字段映射不理解"问题

## 用户画像

* Excel水平：基础（会打开文件、输入数据）

* 可能卡点：安装时的管理员权限提示、字段映射逻辑不理解

* 不需要配图，纯文字描述

## 文档1：安装说明书

### 内容规划

* 篇幅：约1.5页（A4纸）

* 风格：详细但不啰嗦，每步有明确指引

### 章节结构

1. **安装前准备**（系统要求、安装包位置）
2. **安装步骤**（详细步骤，包含管理员权限处理）
3. **安装后验证**（如何确认安装成功）
4. **常见问题**（仅保留安装相关的1-2个问题）

### 详细步骤设计

* 步骤1：找到安装包

* 步骤2：双击运行

* 步骤3：处理管理员权限提示（重点展开）

* 步骤4：选择安装位置

* 步骤5：等待安装完成

* 步骤6：启动软件

## 文档2：使用说明书

### 内容规划

* 篇幅：约3-4页（A4纸）

* 风格：概念解释 + 实例教学结合

### 章节结构

1. **使用流程概览**（整体流程图/列表）
2. **准备工作**（Excel格式要求、示例数据）
3. **详细操作指南**（实例教学方式）

   * 第1步：上传数据

   * 第2步：选择分析模板

   * 第3步：字段映射（重点展开，概念解释+实例）

   * 第4步：时间筛选（可选）

   * 第5步：生成报告
4. **导出结果**（三种导出方式说明）
5. **模板说明**（四个模板简介+适用场景）

### 字段映射重点设计

* 什么是字段映射（概念解释）

* 为什么要做字段映射（必要性说明）

* 实例演示：用一个真实Excel例子，展示如何选择对应列

* 常见问题：智能推荐是什么、选错了怎么办

### 实例教学设计

用一个完整的销售数据例子，从Excel准备到报告生成，全程跟随演示：

* Excel数据示例（3-5行数据）

* 每个步骤的操作说明

* 预期结果展示

## 写作风格要求

### 语言风格

* 口语化，像有人手把手教你

* 避免技术术语，用"大白话"解释

* 关键步骤加粗或标注

### 格式规范

* 使用标题层级清晰划分

* 关键信息用引用块（>）突出

* 步骤用编号列表，便于跟随

### 内容取舍

* 保留：安装步骤、字段映射详解、实例教学

* 删除：系统架构、技术细节、过多原理说明

* 简化：四个模板的详细计算逻辑

## 交付标准

* 安装说明书：用户按步骤操作，能独立完成安装

* 使用说明书：用户按步骤操作，能独立完成一次完整分析

* 两份文档总篇幅控制在5页以内（A4纸）
